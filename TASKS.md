# DevOps Interview Bot - Задачи для разработки

**Проект:** DevOps Interview Bot  
**Дата:** 2026-03-27

---

## 📋 Фаза 1: MVP (Тестовый бот на sandbox)

### 1. Создать базу данных (SQLite schema)
**Приоритет:** 1  
**Статус:** 📋 To Do  
**Описание:**
- Создать SQLite schema для хранения:
  - Задачи (id, category, question, solution, explanation, difficulty, time_limit)
  - Пользователи (user_id, username, created_at)
  - Прогресс (user_id, task_id, solved, time_spent, attempts)
  - Сессии мок-собеса (session_id, user_id, started_at, completed_at, score)

**Файлы:**
- `bot/db/models.py` (SQLAlchemy models)
- `bot/db/crud.py` (CRUD операции)

**Оценка времени:** 1 час

---

### 2. Написать 50 задач с решениями (tasks.json)
**Приоритет:** 2  
**Статус:** 📋 To Do  
**Описание:**
- Создать файл `bot/tasks/tasks.json` с 50 задачами
- Категории:
  - Nginx (10 задач)
  - Bash/CLI (15 задач)
  - Kubernetes (10 задач)
  - Git (10 задач)
  - Docker (5 задач)

**Формат задачи:**
```json
{
  "id": 1,
  "category": "nginx",
  "difficulty": "middle",
  "question": "Найти Top 5 IP адресов из access.log за последние 24 часа",
  "time_limit": 300,
  "solution": "awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -5",
  "explanation": "awk достаёт первый столбец (IP), sort сортирует, uniq -c считает уникальные, sort -rn сортирует по числам в обратном порядке, head -5 выдаёт топ 5",
  "hints": [
    "Используй awk для извлечения IP",
    "Не забудь отсортировать результат"
  ]
}
```

**Оценка времени:** 3 часа

---

### 3. Реализовать режим тренировки (раз в час → задача)
**Приоритет:** 3  
**Статус:** 📋 To Do  
**Описание:**
- Создать handler для отправки задачи раз в час
- Пользователь получает:
  - Вопрос
  - Время на решение (5 мин)
  - Кнопки: "✅ Готов показать решение", "🤔 Нужна подсказка", "⏭️ Пропустить"
- После ответа пользователя:
  - Показать эталонное решение
  - Объяснение
  - Время решения
  - Прогресс (X/50 задач)

**Файлы:**
- `bot/handlers/training.py`
- `bot/utils/cron.py`

**Оценка времени:** 2 часа

---

### 4. Реализовать режим мок-собеса (диалог 30 мин)
**Приоритет:** 4  
**Статус:** 📋 To Do  
**Описание:**
- Создать handler для симуляции собеседования
- Длительность: 30 минут
- Количество задач: 3-5 (в зависимости от скорости решения)
- В конце:
  - Результат (X/Y задач решены)
  - Среднее время на задачу
  - Рекомендации (что прокачать)
  - Детальный feedback

**Файлы:**
- `bot/handlers/mock_interview.py`
- `bot/utils/scoring.py`

**Оценка времени:** 3 часа
