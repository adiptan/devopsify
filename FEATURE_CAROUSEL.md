## 🎯 Проблема

**Сейчас:**
- `/learning` → выбор темы → 1 лекция (200-300 слов) → сразу задачи
- Если тема сложная (например, Nginx) → одной карточки мало
- Нет возможности полистать теорию перед тестами

**Хочется:**
- 10-15 карточек по теме (от простого к сложному)
- Навигация: ⬅️ Предыдущая / ➡️ Следующая
- Быстрый переход: `/nginx 5` → 5-я карточка

### 2. Навигация

**Кнопки:**
```
[⬅️ Предыдущая] [1/10] [➡️ Следующая]
[✅ Начать тесты] [🏠 В меню]
```

**Логика:**
- Первая карточка: `[⬅️]` неактивна
- Последняя карточка: `[➡️]` неактивна
- Счётчик: `[3/10]` — показывает где ты
- `[✅ Начать тесты]` — переход к задачам по теме

---

### 3. Быстрый переход

**Команды:**
- `/nginx 5` → 5-я карточка Nginx
- `/bash 10` → 10-я карточка Bash
- `/k8s 3` → 3-я карточка Kubernetes
- `/git 7` → 7-я карточка Git
- `/docker 4` → 4-я карточка Docker

**Алиасы:**
- `/kubernetes` = `/k8s`

**Валидация:**
- Если номер > количества карточек → "Всего 10 карточек, переход на последнюю"
- Если номер < 1 → "Минимальный номер: 1"

---

### 4. Сохранение прогресса

**БД (users таблица):**
- `learning_topic` (VARCHAR) — текущая тема (nginx/bash/k8s/git/docker)
- `learning_card` (INTEGER) — номер карточки (1-15)

**Логика:**
- При переходе на следующую карточку → сохранить `learning_card`
- При возврате в `/learning` → продолжить с сохранённой карточки
- Кнопка `[🔄 Начать сначала]` → сбросить `learning_card = 1`

---

### 5. Интеграция с задачами

**После последней карточки:**
```
🎓 Поздравляю! Ты прошёл всю теорию по Nginx!

[✅ Начать тесты] [🔄 Пройти снова] [🏠 В меню]
```

**Задачи:**
- Берутся из `tasks.json` по категории (`category: "nginx"`)
- 2-3 задачи (как сейчас)
- После решения → детальное объяснение (как сейчас)

## 📝 Вопросы

1. **Сколько карточек по теме?**
   - Предложение: 10-15 (чтобы не перегружать)
   - Nginx, K8s, Git, Docker: 10 карточек
   - Bash: 15 карточек (больше команд)

2. **Формат карточки:**
   ```
   📘 Nginx: 3/10 — Конфигурация nginx.conf
   
   [Текст 200-300 слов]
   
   Пример:
   ```bash
   sudo nano /etc/nginx/nginx.conf
   ```
   
   Ключевые моменты:
   • server { } — виртуальный хост
   • location { } — обработка URL
   • upstream { } — backend серверы
   
   [⬅️ Предыдущая] [3/10] [➡️ Следующая]
   [✅ Начать тесты] [🏠 В меню]
   ```

3. **Что делать при `/learning` если уже прошёл тему?**
   - Вариант 1: Продолжить с последней карточки
   - Вариант 2: Показать "Пройдено ✅" + кнопка [🔄 Пройти снова]
   - **Предложение:** Вариант 2 (чтобы не путать)
